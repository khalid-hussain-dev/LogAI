# Documentation

This folder contains project reference and evaluation material that should stay separate from the active application code.

## Contents

- `FYP Proposal.pdf`: original project proposal and scope
- `GitHub Repositories.txt`: reference links for the related repositories
- `ANOMALY_MODEL_FLOW.md`: Phase 6 anomaly-detection implementation notes
- `ARCHITECTURE.md`: current implemented system architecture diagram
- `FINAL_CHECKLIST.md`: final demo, verification, and presentation checklist
- `EASY_RUN_GUIDE.md`: simple end-to-end setup and run steps
- `TESTING_GUIDE.md`: practical testing steps before the final demo
- `RENDER_DEPLOYMENT.md`: Render cloud deployment step-by-step instructions

## Usage

- Use the proposal for requirement tracking and evaluation write-ups.
- Use the repository list as supporting reference material.
- Keep implementation work in the active app folders at the workspace root.
